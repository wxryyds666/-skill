# 思维蒸馏框架

> 从原始资料中提取思维模式的方法论。蒸馏的是**怎么想**，不是**知道什么**。
> 知识可以随时上网搜，思维框架才是蒸馏的目标。

---

## 一、核心原则：思维蒸馏 vs 知识提取

| | 知识提取 | 思维蒸馏 |
|------|---------|---------|
| 目标 | 记住事实、数据、信息 | 提取思维框架、判断逻辑、表达模式 |
| 产出 | 陈述性知识（"茅台PE 35倍"） | 程序性思维（"看公司先看护城河"） |
| 存储 | 不需要存（随时搜） | 永久写入记忆文件 |
| 复用 | 原样回忆 | 用同样的方式思考新问题 |
| 例子 | "他说的宁德时代Q3营收增长10%" | "他分析公司总是从供给端切入，不怎么看需求端" |

**检验标准**：蒸馏结果能不能让你用这个人的方式思考一个他没见过的新问题？能 → 蒸馏成功。不能 → 可能只提取了知识。

---

## 二、心智模型识别的三重验证

一个思维模式要被认定为「心智模型」（而非随口一说），必须通过三重验证。

### 验证1: 跨域复现

同一个思维框架出现在至少 2 个不同话题/领域/场景中。

**例**：一份研报里反复出现「从供给端切入分析」——
- 分析光伏行业时，第一反应是查产能数据
- 分析养猪行业时，关注的是出栏量和能繁母猪
- 分析芯片行业时，看的是晶圆厂产能利用率
→ 跨 3 个领域复现 → 这是真正的心智模型

**反例**：只在分析新能源时提到「技术迭代速度是核心变量」，其他行业分析中从未出现 → 只是一次性观察，不是稳定模式。

### 验证2: 有生成力

用这个模型可以推断此人对新问题的可能立场。

**例**：如果「供给端视角」是心智模型——
- 问他怎么看 AI 算力行业 → 他第一反应会是查 GPU 产能、芯片供应
- 问他怎么看咖啡连锁 → 他第一反应是看门店扩张速度和供应链
→ 能生成新推断 → 这是可运行的框架

**反例**：「这个行业值得关注」→ 无法推断他对其他行业的看法 → 没有生成力。

### 验证3: 有排他性

不是所有聪明人都会这样想，这个模式体现了此人的独特视角。

**例**：
- 「从供给端切入」——大部分分析师先看需求增速 → 有排他性 ✓
- 「看公司的护城河」——所有价值投资者都这样 → 没有排他性 ✗
- 「重视管理层诚信」——所有人都这样说 → 没有排他性 ✗

### 判定结果

| 通过几重 | 判定 | 存储方式 |
|---------|------|---------|
| 三重全过 | **心智模型**（核心思维框架） | 写入 my-thinking.md 心智模型区 |
| 过 1-2 重 | **决策启发式**（特定场景的判断规则） | 写入 my-thinking.md 决策启发式区 |
| 全不过 | 随口说的，不纳入 | 丢弃 |

**宁少勿多。3 个深刻的心智模型远比 10 个浅薄的原则有价值。**

## 三、从资料中提取思维模式的操作步骤

### Step 1: 通读全文，标记候选模式

读完资料时，边读边问自己：
- 这个人反复提到什么？≥3 次出现的话题/角度/方法 → 候选
- 这个人用什么框架在分析问题？他先看什么、后看什么、不看什么？
- 这个人怎么下结论的？他的判断依据是什么？
- 这个人的观点里，什么跟主流不一样？

### Step 2: 列出候选清单

把观察到的候选思维模式列出来。通常会得到 5-15 个候选。

### Step 3: 三重验证筛选

对每个候选执行三重验证：
1. 在资料里找 ≥2 个不同话题的证据（跨域复现）
2. 试着用它推断一个资料里没提的问题（有生成力）
3. 判断是不是所有聪明人都这样想（排他性）

### Step 4: 分级存储

根据验证结果分级写入记忆。

## 四、表达 DNA 的提取维度

从资料中分析这个人的说话方式，不是为了模仿，而是为了理解他的表达模式。

### 句式偏好
- 长句（>30 字）vs 短句（<15 字）？
- 爱用疑问句还是陈述句？
- 先给结论还是先铺垫过程？
- 段落多长？一句一段还是大段论述？

### 词汇特征
- 高频词 / 口头禅
- 自创术语或专属概念
- 从不使用的词（禁忌词）
- 用口语词还是书面语？

### 节奏感
- 怎么转折？（"但是"、"不过"、"说白了"）
- 怎么强调？（重复、反问、对比）
- 怎么收尾？

### 确定性表达
- 「我觉得」「应该是」「可能是」→ 留余地型
- 「一定是」「显然」「毫无疑问」→ 断言型
- 「不好说」「看情况」→ 审慎型

### 幽默方式
- 自嘲？讽刺？荒诞？冷幽默？不幽默？
- 用例子还是用比喻来搞笑？

### 引用习惯
- 爱引谁？（巴菲特？达利欧？自己的经历？）
- 引用数据还是引用故事？

## 五、矛盾处理原则

矛盾是人格的核心特征，不是需要修复的 Bug。

### 三种矛盾类型

**1. 时间性矛盾**（观点演化）
- 早期说 A，后来说 B
- 处理：都记录，标注时间，以近期为主
- 例：「2023 年他说新能源车是泡沫，2025 年他开始重仓比亚迪」

**2. 领域性矛盾**（不同场景不同规则）
- 股票上主张 X，生活上主张 Y
- 处理：分领域记录，不强求统一
- 例：「投资上极度保守（只买龙头），但生活上很大胆（经常尝试新事物）」

**3. 本质张力**（价值观内在冲突）
- 既追求 A 又追求 B
- 处理：记录为核心张力，这是深度来源
- 例：「既追求确定性（只买看得懂的），又渴望高回报（经常研究高成长股）」

### 错误处理
- ❌ 选一边忽略另一边
- ❌ 编一个调和解释
- ❌ 假装矛盾不存在
- ✅ 如实记录，标注类型，让矛盾共存

## 六、记忆写入格式

### 心智模型写入格式（写入 my-thinking.md）

```markdown
### 心智模型：[名称]
- **一句话**：[最简洁的描述]
- **证据**：
  - [场景1]：[来源文件]，表现：...
  - [场景2]：[来源文件]，表现：...
- **可推断**：[如果用这个模型问一个新问题，他会怎么想]
- **局限性**：[这个模型在什么情况下可能失效]
- **蒸馏时间**：[YYYY-MM-DD]
```

### 决策启发式写入格式（写入 my-thinking.md）

```markdown
### 启发式：[规则名]
- **规则**：如果 [X]，则 [Y]
- **场景**：[什么时候用]
- **证据**：[来源文件]
- **蒸馏时间**：[YYYY-MM-DD]
```

### 表达 DNA 写入格式（写入 my-style.md）

```markdown
### [维度名称]
- 观察：[具体表现]
- 示例：[原文片段]
- 蒸馏时间：[YYYY-MM-DD]
```

## 七、蒸馏质量自检

每次蒸馏完成后，问自己：

### 思维提取
- [ ] 提取出的模式是「怎么想」，不是「知道什么」？
- [ ] 每个心智模型有三重验证支撑？
- [ ] 区分了心智模型（L3）和决策启发式（L2）？
- [ ] 矛盾被保留而非抹平？

### 表达提取
- [ ] 分析的是「怎么说的」，不只是「说了什么」？
- [ ] 抓到了辨识度特征，不是泛泛的「喜欢用短句」？

### 记忆写入
- [ ] 心智模型写入了 my-thinking.md？
- [ ] 决策启发式写入了 my-thinking.md？
- [ ] 表达 DNA 写入了 my-style.md？
- [ ] 训练记录写入了 training-log.md？

### 持久性
- [ ] 写入是追加模式（不覆盖已有内容）？
- [ ] 每次写入标了蒸馏时间？
- [ ] 新旧内容不冲突？（如有冲突用矛盾处理原则）

## 八、常见陷阱

| 陷阱 | 表现 | 纠正 |
|------|------|------|
| **知识伪装成思维** | 蒸馏出"宁德时代Q3营收涨了10%" | 问：这是思考方式还是事实？事实就删掉 |
| **过度提取** | 蒸馏出了15个心智模型 | 三重验证砍掉凑数的 |
| **通用化** | 蒸馏出"要看基本面"这种所有人都说的话 | 没有排他性的降级或丢弃 |
| **抹平矛盾** | 强行统一不一致的观点 | 矛盾是深度，保留 |
| **假深度** | 把简单的观点包装成深奥的模型 | 用大白话描述，看还有没有内容 |
| **不写入** | 蒸馏完只在脑子里记住，没写进文件 | **必须写入记忆文件，否则下次启动丢失** |
